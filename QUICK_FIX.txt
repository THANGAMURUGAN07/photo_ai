PROBLEM: Downloads work on your computer but not on other devices

SOLUTION: Update your .env file with your network IP

┌────────────────────────────────────────────────────────────────┐
│ STEP 1: Update .env file                                       │
└────────────────────────────────────────────────────────────────┘

Open .env and change:
   FROM: BASE_URL=http://localhost:5000
   TO:   BASE_URL=http://192.168.29.182:5000

Your network IP: 192.168.29.182

┌────────────────────────────────────────────────────────────────┐
│ STEP 2: Restart your server                                   │
└────────────────────────────────────────────────────────────────┘

Press Ctrl+C to stop the server, then run:
   node app.js
   (or npm start)

┌────────────────────────────────────────────────────────────────┐
│ STEP 3: Check Windows Firewall                                │
└────────────────────────────────────────────────────────────────┘

1. Open Windows Defender Firewall
2. Click "Allow an app through firewall"
3. Make sure Node.js is allowed on Private networks

┌────────────────────────────────────────────────────────────────┐
│ STEP 4: Test from another device                              │
└────────────────────────────────────────────────────────────────┘

On another device (phone/tablet), open browser and go to:
   http://192.168.29.182:5000

If you see PhotoFlow, downloads will work!
